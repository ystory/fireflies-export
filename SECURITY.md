# Security Policy

## Supported versions

The latest published `0.x` release is supported for security fixes. Older `0.x` releases may receive fixes at maintainer discretion, but support is not guaranteed.

## Reporting a vulnerability

Please do not report security vulnerabilities in public GitHub issues.

Preferred channels:

1. GitHub private vulnerability reporting, when enabled for this repository
2. Email [oss@ystory.kr](mailto:oss@ystory.kr) with the subject line `fireflies-export security report`

Please include:

- A clear description of the issue
- Steps to reproduce or a proof of concept
- Affected versions or commit range, if known
- Any suggested mitigation or workaround

You will receive an acknowledgment as soon as practical. We will work toward a fix, coordinated disclosure, and a release once the issue is understood.
